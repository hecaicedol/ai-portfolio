PROJECT 01 / 06 ● Functional

Self-Healing Multi-Agent Pipeline

Constitutional AI critic · Bounded reflection · pgvector episodic memory

The problem

In production, the most expensive AI failures are silent: a confident wrong invoice number propagates, a downstream system trusts the bad output, the wrong vendor gets paid. Standard pipelines treat the LLM as a black box — if it returns something, the pipeline returns "success."

This pipeline does the opposite: every output is adversarially evaluated by a critic, failures trigger a bounded reflection loop, and the system maintains an episodic memory of past errors that the critic consults before every evaluation.

Key architectural decisions
LangGraph orchestrationExplicit, inspectable state transitions over LangChain's opaque chains
Constitutional AI criticPer-principle scoring (4 axes) yields actionable feedback; single-rubric LLM-as-judge does not
Episodic memory in pgvectorDomain-specific failure patterns surface across thousands of past runs — reuses the Postgres we already need
Hard cap: 3 reflection iterationsUnbounded retry loops are a top-3 incident class for agents. Cap forces loud failures over silent token spend
Lessons learned the hard way
  • Critics need memory, not just rubrics. A stateless critic re-makes the same mistake every run.
  • Constitutional principles must be evaluable, not philosophical. "Output should be helpful" is unscorable; "every required field has a non-empty value matching its type" is.
  • Reflection loops without hard caps burned $40 of credits on a single malformed PDF in week one.

Enterprise Knowledge OS · GraphRAG

The problem

PROJECT 03 / 06 ◐ Scaffolded

Autonomous Research Agent · MemGPT pattern

Three-tier memory · Persistent state · Tavily + arXiv + PDF reports

The problem

A research agent that "forgets" between sessions is a chatbot, not a researcher. The MemGPT paper showed that explicit memory tiering — working / episodic / semantic — with a controller that promotes and evicts beats relying on bigger context windows.

Key architectural decisions
Three-tier memoryWorking (in-context) / episodic (recent runs) / semantic (consolidated facts). Each tier has explicit promotion rules.
Memory controller as a nodePromotion and eviction are first-class graph nodes — inspectable in Langfuse, not buried in a hook
arXiv + Tavily + scrapingMix of structured (arXiv API) and unstructured (Tavily) sources reduces source-bias in synthesized reports

PROJECT 04 / 06 ◐ Scaffolded

Multi-Agent Debate System · Society of Mind

5 specialized agents · 3-round protocol · WebSocket streaming · Consensus measurement

The problem

Single-agent reasoning collapses to the model's prior. Five agents with distinct, persistent perspectives — Optimist, Skeptic, Financial Analyst, Risk Officer, Devil's Advocate — generate dissent and surface tradeoffs that a single chain-of-thought hides.

Key architectural decisions
5 roles, not 2Two-agent "debate" collapses to majority. Five roles, structured three rounds, prevent groupthink
Three-round protocolOpening · rebuttal · synthesis. Bounded, measurable, prevents infinite loops
Consensus score per roundCosine similarity of agent positions over rounds. Measurable convergence, not vibes
WebSocket streamingEach agent's tokens stream live to UI — debate is a watchable artifact, not a black box

Hardcoded workflows break the moment the business changes. A meta-agent that generates the DAG from natural language and executes via Model Context Protocol servers turns any new business request into a runnable pipeline without engineering work.

PROJECT 06 / 06 ◐ Scaffolded

Agentic RAG · Vector DB Benchmark

pgvector vs Qdrant vs Pinecone · Contextual retrieval · RRF hybrid · Auto-optimizer agent

The problem

Most RAG implementations stop at "we chose Pinecone." This project asks the harder question: which one actually performs better on your corpus, and by how much? Three vector stores run in parallel, all queries dual-written, all metrics tracked.

Key architectural decisions
Three stores in parallelSame corpus, same queries, same eval. Real apples-to-apples — published benchmarks rarely are
Anthropic's contextual retrieval~49% retrieval failure reduction by prepending chunk context. Free quality lift before any reranker
Hybrid: BM25 + vector + rerankRRF fusion of lexical + semantic + Cohere rerank. Each stage measured independently
Auto-optimizer agentWatches Ragas scores; when faithfulness drops, tunes chunk size / rerank-k / fusion weights, A/B's against held-out set
What this proves to hiring teams
  • I measure things instead of citing them.
  • I know that "use a vector DB" is the easy half of RAG — the chunking, fusion, and eval loop is the hard half.
  • I treat optimization as a closed loop, not a one-off tuning session.
